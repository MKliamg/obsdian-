## 什么是双链笔记

双链笔记（Bidirectional Linking）是指笔记之间可以**相互引用**的系统。当笔记 A 链接到笔记 B 时，笔记 B 也会自动显示"谁链接到了我"。

### 传统笔记 vs 双链笔记

```
传统笔记（单向）        双链笔记（网状）
    A                      A ←→ B
    ↓                      ↑   ↓
    B                      D ←→ C
    ↓                        ↘ ↗
    C                         E
```

### 为什么双链重要

| 优势 | 描述 |
|------|------|
| 🔍 **知识发现** | 通过链接发现意想不到的联系 |
| 🧠 **模拟思维** | 像大脑神经元一样连接概念 |
| 📚 **渐进积累** | 新笔记自动与旧笔记产生联系 |
| 🎯 **减少遗忘** | 多次复习关联内容 |

## 创建链接

### 基本链接语法

```markdown
# 方式1：Wiki 链接（推荐）
[[笔记名称]]

# 方式2：Markdown 链接
[显示文字](笔记名称.md)
```

### 链接类型详解

#### 1. 基础链接

```markdown
今天学习了 [[Python基础]]，感觉很有趣。
```

#### 2. 别名链接

```markdown
# 使用 | 分隔链接和显示文字
参考 [[Python基础|Python 入门教程]]

# 效果：显示 "Python 入门教程"，链接到 "Python基础"
```

#### 3. 标题链接

```markdown
# 链接到特定标题
详见 [[Python基础#函数定义]]

# 链接到当前笔记的标题
跳转到 [[#创建链接]]
```

#### 4. 块链接

```markdown
# 在目标段落末尾添加块 ID
这是一个重要的定义。 ^definition

# 在其他笔记中引用
参考 [[Python基础#^definition]]
```

### 快速创建链接

| 方法 | 操作 |
|------|------|
| 输入 `[[` | 自动弹出笔记列表 |
| 选中文字 + `[[` | 将选中文字变为链接 |
| `Ctrl/Cmd + K` | 快速添加链接 |

### 创建不存在的笔记

```markdown
[[这个笔记不存在]]
```

点击这个链接会自动创建新笔记。这是一种**自底向上**的笔记创建方式。

## 反向链接

反向链接（Backlinks）显示**哪些笔记链接到当前笔记**。

### 查看反向链接

1. 打开右侧边栏
2. 点击"反向链接"面板
3. 查看链接到当前笔记的所有笔记

### 反向链接面板

```
┌──────────────────────────────┐
│ 反向链接                      │
├──────────────────────────────┤
│ 链接提及 (3)                  │
│ ├─ 📄 学习笔记                │
│ │   ...关于 [[当前笔记]] 的... │
│ ├─ 📄 项目计划                │
│ │   ...参考 [[当前笔记]]...    │
│ └─ 📄 读书笔记                │
│     ...详见 [[当前笔记#章节]]  │
├──────────────────────────────┤
│ 未链接提及 (1)                │
│ └─ 📄 会议记录                │
│     ...提到了"当前笔记"...     │
└──────────────────────────────┘
```

### 链接提及 vs 未链接提及

| 类型 | 描述 |
|------|------|
| **链接提及** | 使用 `[[]]` 明确链接的笔记 |
| **未链接提及** | 仅提到笔记名称但未创建链接 |

> **技巧**：可以将"未链接提及"转换为正式链接，增强知识网络。

## 图视图

图视图（Graph View）将你的笔记网络可视化为一张**知识图谱**。

### 打开图视图

- 快捷键：`Ctrl/Cmd + G`
- 或点击左侧边栏的图谱图标
- 或使用命令面板：`Ctrl/Cmd + P` → 搜索"图"

### 图视图结构

```
       ○ 笔记A
      /|\
     / | \
    /  |  \
   ○   ○   ○
笔记B 笔记C 笔记D
   \   |   /
    \  |  /
     \ | /
       ○
     笔记E
```

- **节点**：每个圆点代表一篇笔记
- **连线**：表示笔记之间的链接
- **大小**：节点大小反映链接数量
- **颜色**：可按文件夹或标签着色

### 图视图设置

| 设置 | 描述 |
|------|------|
| 过滤器 | 按路径、标签、文件类型筛选 |
| 分组 | 按颜色分组显示 |
| 显示选项 | 显示/隐藏标签、附件、孤立节点 |
| 力度设置 | 调整节点间距和斥力 |

### 局部图视图

在笔记中只查看与当前笔记相关的图：
1. 右键点击标签页
2. 选择"打开局部图"
3. 或使用命令面板搜索"局部图"

## 链接策略

### 何时创建链接

**应该链接**：
- ✅ 概念之间有关联
- ✅ 想要在未来找到这个联系
- ✅ 两个主题经常一起出现

**不需要链接**：
- ❌ 仅仅因为提到了某个词
- ❌ 链接不会增加任何价值
- ❌ 过度链接造成混乱

### 链接类型

#### 1. 概念链接

```markdown
[[人工智能]] 的一个重要分支是 [[机器学习]]。
[[机器学习]] 又包含 [[深度学习]] 和 [[强化学习]]。
```

#### 2. 引用链接

```markdown
这个观点来自 [[《思考快与慢》]]，作者是 [[丹尼尔·卡尼曼]]。
```

#### 3. 结构链接

```markdown
## 相关笔记
- [[上级主题]]
- [[同级主题1]]
- [[同级主题2]]
- [[子主题1]]
```

#### 4. 时间链接

```markdown
继续 [[2024-01-14]] 的讨论...
这与 [[Q1目标]] 相关。
```

### MOC（Map of Content）

MOC 是一种特殊的笔记，用于组织和导航相关笔记：

```markdown
# 编程学习 MOC

## 编程语言
- [[Python]]
- [[JavaScript]]
- [[Go语言]]

## 开发工具
- [[Git]]
- [[VS Code]]
- [[Docker]]

## 方法论
- [[敏捷开发]]
- [[测试驱动开发]]

## 项目
- [[个人网站项目]]
- [[自动化脚本集]]
```

## 最佳实践

### 1. 原子化笔记

每篇笔记只关注**一个概念**：

```markdown
# 好的做法
[[复利]] - 只讲复利概念
[[72法则]] - 只讲这个估算法则
[[投资回报率]] - 只讲 ROI

# 而不是
[[财务概念大全]] - 包含所有内容
```

### 2. 链接时添加上下文

```markdown
# 好的做法
[[GTD方法]] 强调将任务从大脑中"清空"到可信的系统中。

# 不好的做法
参考 [[GTD方法]]。
```

### 3. 定期审查反向链接

- 每周查看笔记的反向链接
- 发现新的联系
- 将"未链接提及"转换为正式链接

### 4. 不要害怕孤立笔记

```
新笔记开始时可能没有链接，
随着积累，联系会自然产生。
         ○          ○──○
        新笔记  →   ╱│╲
                  ○ ○ ○
                   网络化
```

### 5. 使用别名保持可读性

```markdown
# 可读性更好
根据 [[费曼|理查德·费曼]] 的观点...

# 而不是
根据 [[理查德·费曼]] 的观点...
```

## 常见问题

### Q1: 链接太多会不会很乱？

不会，只要链接有意义。质量比数量重要。图视图可以帮助你发现过度链接的区域。

### Q2: 笔记改名后链接会断吗？

不会！Obsidian 会自动更新所有链接。

### Q3: 如何处理同名笔记？

使用完整路径：
```markdown
[[文件夹/笔记名称]]
```

或者给笔记起唯一的名称。

### Q4: 链接到不存在的笔记好吗？

非常好！这是一种"先规划后填充"的方法。空链接提醒你以后要写这篇笔记。

### Q5: 如何找到孤立笔记？

在图视图中：
1. 开启"孤立节点"显示
2. 查找没有连接的节点
3. 考虑添加链接或删除无用笔记

## 练习任务

试着完成以下练习：

1. **创建3篇相互链接的笔记**
   - 笔记A：一个主题
   - 笔记B：相关概念
   - 笔记C：应用实例

2. **查看反向链接**
   - 打开任意一篇笔记
   - 查看右侧反向链接面板
   - 尝试将"未链接提及"转换为链接

3. **探索图视图**
   - 打开全局图视图
   - 尝试不同的显示设置
   - 发现你知识网络的结构
